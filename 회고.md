## 🎓 주요 학습 내용

### 기술적 학습
1. **TDD 사이클**: Red → Green → Refactor
3. **Test Fixture**: 재사용 가능한 테스트 데이터
4. **Testcontainers**: 실제 환경과 동일한 테스트

### 사고방식 변화
1. "어떻게 구현할까?" → **"어떻게 테스트할까?"**
2. "동작하는 코드" → **"검증 가능한 코드"**
3. "빠른 개발" → **"안정적인 개발"**

---

##  느낀점
1. TDD를 적용하면서 어려웠던 점은 Red → Green → Refactor의 과정을 온전하게 따라갈 수는 없었다는 것입니다. 테스트 코드를 작성할 때 시나리오만을 가지고 아직 존재하지 않는 코드의 테스트 코드를 작성한다는 것이 까다로운 문제였다고 생각합니다.

2. 테스트 코드가 가이드라인이 되면서 개발을 진행할 때에 이슈 사항을 체크하기가 굉장히 쉬웠던 것 같습니다. 테스트 코드를 먼저 작성하는 오버헤드는 분명히 존재하지만 그에 따라 돌아오는 리턴값이 충분히 의미있었다고 생각합니다.

3. 테스트 코드를 작성하는 방법과 테스트 코드 내에서의 책임 분리에 대해서 좀 더 많이 깨닫게 된 시간이었다고 생각합니다.
